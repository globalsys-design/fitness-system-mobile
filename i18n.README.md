# Internationalization (i18n) Setup

This application is configured for multi-language support using `next-intl`.

## Configuration Files

- `lib/i18n.config.ts` - Centralized locale settings (supported languages, defaults)
- `i18n.request.ts` - next-intl request configuration for loading messages
- `messages/` - Translation files (pt-BR, en-US, es-ES)

## Supported Languages

- **pt-BR** (Portuguese - Brazil) - Default
- **en-US** (English - United States)
- **es-ES** (Spanish - Spain)

## Installation

To enable i18n features, install the required dependency:

```bash
npm install next-intl
# or
pnpm add next-intl
```

## Using Translations

### In Server Components

```typescript
import { getTranslations } from 'next-intl/server';

export default async function Page() {
  const t = await getTranslations('namespace');
  
  return <h1>{t('key')}</h1>;
}
```

### In Client Components

```typescript
'use client';

import { useTranslations } from 'next-intl';

export function MyComponent() {
  const t = useTranslations('namespace');
  
  return <h1>{t('key')}</h1>;
}
```

## Adding New Translations

1. Add the key and English translation to `messages/en-US.json`
2. Add the key and Portuguese translation to `messages/pt-BR.json`
3. Add the key and Spanish translation to `messages/es-ES.json`

Keep the structure consistent across all files.
